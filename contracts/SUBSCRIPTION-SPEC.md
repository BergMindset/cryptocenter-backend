# ТЗ: контракт подписки PoolsSubscription v2 ($20/мес, USDC, Base)

> Статус: ДРАФТ ДЛЯ НЕЗАВИСИМОГО АУДИТА (v2 — после внутренней адверсариальной панели 17.07.2026:
> 3 аудитора-агента, 19 находок → 6 must-fix внесены в код). Не деплоить до аудита.
> Деплой, владение (owner) и казна — зона основателя. Сервис ключей не держит.

## 1. Что это
Абонентская плата за сервис «вход в пулы по схеме фонда» на cryptocenter.finance.
Пользователь подписывается ОДИН раз (одна транзакция; для обычных кошельков и для
смарт-кошельков ERC-1271 — со встроенным permit), дальше $20 списываются автоматически
раз в 30 дней, пока он не отменит подписку или не отзовёт разрешение.

## 2. Экономика и параметры деплоя
- Токен: USDC на Base `0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913` (6 decimals; FiatTokenV2_2 —
  EIP-2612 permit в двух формах: ECDSA (v,r,s) и bytes-подпись с поддержкой ERC-1271).
- price = 20_000000 (=$20). PERIOD = 30 дней. RENEW_WINDOW = 1 день. RENEW_GRACE = 3 дня. TREASURY_DELAY = 48 ч.
- MAX_PRICE (иммутабельный потолок): рекомендуем 50_000000 (=$50).
- treasury: адрес казны фонда — ВЫДАЁТ ОСНОВАТЕЛЬ, строго из Центра, побайтовая сверка (address poisoning!).
- owner: кошелёк/мультисиг основателя (Ownable2Step; renounceOwnership ЗАБЛОКИРОВАН).

## 3. Инварианты безопасности (что проверять аудитору)
1. **Контракт не хранит средств**: `safeTransferFrom(user → treasury)` в той же транзакции.
2. **Снапшот цены (priceCap)**: юзер платит не больше цены, действовавшей на момент ЕГО подписки;
   повышение `price` владельцем действует только на новые/повторные подписки. MAX_PRICE ограничивает
   цену за период; реальный потолок ущерба юзера в любом сценарии (включая компрометацию owner) =
   выданный им allowance. Инвариант фронта: кап permit строго 12 × price, НИКОГДА unlimited.
3. **charge() permissionless, но безвреден**: выгоды вызывающему нет, средства только в treasury.
   Инвариант: нельзя оплатить больше одного PERIOD вперёд (`block.timestamp + RENEW_WINDOW ≥ paidUntil`).
   Пред-окно RENEW_WINDOW (1 день) закрывает разрыв доступа между истечением и суточным кроном;
   при раннем продлении base = paidUntil (двойной оплаты времени нет).
4. **Без долгов**: просрочка ≤ grace — продление встык; дольше — от `block.timestamp`.
5. **permit фронтраннинг**: `try/catch` + явная проверка `allowance ≥ price` (честный `PermitFailed`
   вместо загадочного реверта). Смарт-кошельки (Coinbase Smart Wallet и др.) — bytes-форма permit.
6. **cancel() и resume() всегда доступны** (не под pause). После cancel юзер может передумать —
   `resume()` (событие Resumed обязано попадать в индексацию крона).
7. **Смена treasury — только через таймлок 48ч**: proposeTreasury → (48ч, событие TreasuryProposed =
   алерт в TG) → executeTreasury. Мгновенной смены нет.
8. **Pause** останавливает subscribe/charge, но НЕ cancel/resume/isActive. renounceOwnership запрещён
   (renounce при паузе = вечный кирпич).
9. USDC не fee-on-transfer, не ERC-777; ReentrancyGuard — защита в глубину. uint64/uint96 касты
   безопасны (paidUntil ~584 млрд лет; MAX_PRICE проверяется ≤ uint96.max в конструкторе).

## 4. UX подписки (инварианты фронта)
1. Кнопка «Подписаться — $20/мес» → EIP-712 подпись permit: кап = 12 × price ($240), deadline ≤ 1 час.
   Домен permit брать ТОЛЬКО через `eip712Domain()` (ERC-5267) в рантайме — у Base-USDC version = "2",
   никаких хардкодов домена.
2. Обычный кошелёк → `subscribeWithPermit(value, deadline, v, r, s)`; смарт-кошелёк (ERC-1271) →
   `subscribeWithPermit(value, deadline, bytes signature)`. Fallback без permit: approve($240) + `subscribe()`.
3. Отмена: `cancel()` + рекомендация «отозвать разрешение» (approve 0). Вернуться: `resume()`.
4. Если `allowance > 0 && paidUntil == 0` (висящая подпись исполнена без подписки) — фронт показывает
   баннер «отозвать разрешение».
5. Напоминания об истечении: за 5 дней и в день (сайт + TG).

## 5. Продление (крон) — зона основателя (паттерн berg-signer) или Gelato/Chainlink Automation
- Ежедневно: индексация событий Subscribed/Charged/Cancelled/**Resumed** → список активных →
  для тех, у кого `paidUntil − now ≤ RENEW_WINDOW`, вызвать `charge(user)`.
- Перед вызовом — multicall pre-check `allowance + balanceOf` (не жечь газ об заведомый реверт).
- Индексация: `lastProcessedBlock` + getLogs-backfill + суточная полная сверка с чейном.
- Газовый ключ: одноразовый, только ETH на газ (~$5), НИКАКОГО доступа к средствам.
- Неудачное списание: подписка спит, доступ гаснет по paidUntil, долги не копятся.

## 6. Интеграция с сервисом
- Гейт: бэкенд читает `isActive(wallet)` перед POST /api/pools/plan. Кэш 60 сек, fail-closed.
- Ранбук (написать до запуска): (а) pause не останавливает «часы» paidUntil — дни компенсируются
  офф-чейн; (б) миграция при блэклисте контракта/казны со стороны Circle; (в) регламент возврата
  за гонку cancel-vs-charge у самого истечения (ущерб = одно продление, в казну).

## 7. Порядок запуска
1. Основатель: адрес treasury (из Центра, побайтовая сверка) + owner-кошелёк.
2. Независимый аудит (включить в пакет 15–25 июля). В код заложен отчёт внутренней панели (19 находок, 6 fixed).
3. Деплой на Base — основатель; верификация исходника на Basescan.
4. Я: гейт isActive + кнопка подписки на /pools (обе формы permit) + индексация событий + ТЗ крона.
5. Параллельно: разовая оплата $9.99/вход по адресу приёма — живёт без контракта.
